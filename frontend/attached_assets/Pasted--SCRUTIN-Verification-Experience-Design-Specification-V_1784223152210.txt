# SCRUTIN — Verification Experience Design Specification (V1)

> This document defines everything the user sees after pressing the Verify button and before the final report appears.

The goal is NOT to create a loading screen.

The goal is to transform waiting into curiosity.

The user should never feel that the system is "thinking."

Instead, the system should feel alive.

Every second should communicate progress, transparency and trust.

---

# 1. Philosophy

Most AI products show

Loading...

Thinking...

Generating...

Scrutin should never use these.

Verification is a process.

The user should experience that process.

Every screen should answer

"What is happening right now?"

instead of

"Please wait."

---

# 2. Overall Interaction

User pastes

↓

Presses Verify

↓

↓

↓

Hero text fades

↓

Verification Workspace expands

↓

Card 01 slides into focus

↓

Cards 02–04 remain stacked behind

↓

Each completed card slides away

↓

Next card enters

↓

Final report appears

Never use page transitions.

Everything should happen inside one continuous canvas.

# Final User Emotion

The user should never think

"I'm waiting."

Instead they should think

"I understand exactly what Scrutin is doing, I discovered useful related information while it worked, and I trust the process because it is transparent."

The waiting time itself becomes part of the product experience.
